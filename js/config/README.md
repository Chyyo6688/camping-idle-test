# 配置文件说明

`js/config/` 保存日常可能调整的静态数据和简单常量。这里的文件依旧按 `index.html` 中的顺序作为普通浏览器脚本加载，不使用 ES Modules。

## 日常配置

- `fishingCookingConfig.js`：鱼类目录、稀有度分组、钓鱼结果概率、菜谱、烹饪奖励、背包与自动烹饪上限。新增鱼或菜谱、调整钓鱼概率和烹饪奖励时修改。改动后重点回归旧存档、鱼类统计、背包容量、钓鱼、烹饪和锦鲤放生动画。
- `economyConfig.js`：篝火升级成本、产出/消耗速率、离线收益时长、建造模式解锁购买数和场景木材上限。适合平衡数值调整；改动后重点回归离线收益、升级、商店解锁、每日重置与旧存档迁移。
- `activityConfig.js`：Camper 自动活动的时长、权重、条件、动作姿势和目标类型。适合调整活动节奏；改动后重点回归自动行动、Action Queue、动画和活动音效。
- `tutorialConfig.js`：首次引导步骤、独立帮助页及首件装备标识。适合修改教程文案和步骤目标；改动后重点回归新存档、旧存档、手机端遮罩与按钮定位。
- `camperConfig.js`：Camper 图层表、外观选项、动作文字、想法、性格和捏脸问卷。适合新增或调整外观及问卷内容；改动后重点回归旧角色档、捏脸、Camper Card、各动作动画和夜间素材。
- `camperProgressionConfig.js`：Camper 长期互动习惯的 trait 权重、每日计入次数、长期修正范围，以及每日占卜到冒险修正的映射。主场景行为不读取这些修正；深山冒险原型会读取它们进行反应和结果加权。
- `adventurePrototypeConfig.js`：深山原型的体力、背包容量、工具/任务物品目录、后续地点记录、12 个事件的反应与实际结算结果。调整后需回归 storage 流转、五事件上限、日志和旧存档补全。
- `sceneConfig.js`：营地点位、不可行走区域和活动目标区域。适合调整场景坐标；改动后重点回归移动寻路、拖拽放置、点击命中、手机端布局和所有自动活动。

## 目录型配置

- `gearCatalog.js`：商店装备、分类、价格、依赖、放置和装备元数据。修改后需完整回归购买、解锁、装备、放置、拖拽、座椅和旧存档。
- `weatherCatalog.js`：天气类型及其每日营地效果。修改后需回归每日重置、旧存档和天气 UI。
- `soundJournalCatalog.js`：声音图鉴条目、音频路径、解锁和循环属性。修改后需回归浏览器自动播放限制、长循环、混音、静音和声音存档。
- `divinationCatalog.js`：塔罗牌、问题及结果文本。修改后需回归每日占卜结果、旧存档和塔罗 UI。
- `ichingCatalog.js`：龟壳占卜使用的卦象静态数据。修改后需回归掷卦流程、每日限制和结果显示。
- `seatableFurnitureDefs.js`：可坐家具的派生定义。修改后需回归对应装备的放置、朝向和 Camper 坐姿。

`js/core/`、`js/systems/` 和 `js/camper/` 保存存档、迁移、状态、事件、业务流程和动画运行逻辑，一般不作为日常配置直接修改。尤其不要把状态、DOM 操作、事件绑定或存档迁移放进 `js/config/`。

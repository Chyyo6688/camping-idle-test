# Adventure System Design

最后更新：2026-07-13  
当前阶段：本地多人事件原型完成，进行最终整体回归

## 核心定位

游戏核心仍然是 cozy camping idle。营地是玩家长期停留、经营、布置和照顾 Camper 的地方；冒险是 Camper 带着日常积累短暂出门，自动经历一段有性格、有物品流转、有日志沉淀的小故事。

冒险不应变成复杂任务表或战斗系统。基础感觉是：收拾一个小背包，然后出门看看会发生什么。

## 营地与冒险

营地负责：

- Camper 的日常生活、互动习惯和长期成长。
- 营地经营、布置、钓鱼、烹饪和物资积累。
- 冒险前准备、收藏展示、纪念品存放。
- 未来好友访问与轻互动。

冒险负责：

- 使用 `baseTraits + habitModifiers + dailyAdventureModifiers` 生成符合角色但不固定的反应。
- 使用玩家携带的 5 件以内背包物品产生可见分支。
- 产生实际获得、消耗、丢失、解锁和日志。
- 带回工具、材料、线索、收藏、菜谱或营地内容。

完整循环：

```text
经营营地
→ 选择冒险地图
→ 选择地图路线
→ 整理最多 5 件背包物品
→ 未来可选邀请好友
→ 自动经历最多 5 个事件
→ 获得、消耗或丢失内容
→ 返回营地查看日志
→ 战利品进入对应 Storage
→ 体力随时间恢复
```

## 玩家可见决策

基础冒险出发前只保留简单决策：

- 选择地图。
- 选择路线。
- 整理最多 5 件物品。
- 未来可选邀请好友。

玩家不手动选择冒险目标。当前只有深山时可以暂不显示地图选择，实际准备流程是“选择路线 → 整理背包 → 出发”。不要要求玩家每趟手动选择传统任务目标、逐件设置自动使用权限、配置复杂策略、为每个事件选择答案，或阅读概率与公式。物品使用、事件反应和结果判定由 Camper 与系统自动完成。

界面只展示 Camper 气泡、动作、场景变化、物品获得/丢失/消耗提示、故事化结果和冒险日志。不要展示 trait 数值、权重、成功概率、判定公式、运势修正或调试信息。

## 事件判定来源

事件决策共同读取：

```text
baseTraits
+ habitModifiers
+ dailyAdventureModifiers
+ 携带物品
+ 当前地图状态
+ Camper 长期记忆
+ 本次 eventFlags
+ 随机性
```

反应选择保持多候选加权随机，不固定选择最高 trait。每日运势只影响结果倾向，不覆盖人格。所有会影响事件的物品必须实际携带才生效；仓库中拥有但没带的物品不能自动提供效果。

## 故事三层

表层是每趟最多 5 个事件组成的小故事。日志应能读成一段过程，而不是五条独立系统记录。

地图层是每张地图自己的当地故事，包括当地人物、环境问题、路线变化、小型谜团、连续事件链和长期地图状态。深山当前主题包括吊桥、木屋、旧瞭望塔、护林员痕迹、山林动物和反复出现的白影。

跨地图暗线只通过稀有事件、唯一物品、重复事件变化、不一致日志、相同符号、熟悉感、错误记忆、短暂残影或未来多人视角差异慢慢出现。不要做主线任务按钮、章节进度条、强制剧情列表或大段世界观解释。

## 内部暗线：“回途”

“回途”是内部设计名，不是前期玩家应直接知道的名词。它连接的不只是地理位置，也可能连接不同时间的痕迹、没有被选择的路线、其他可能发生过的结果、不同队伍留下的记忆与物品，以及同一个 Camper 的不同经历残影。

前期只表现为难以解释但似乎有关的细节，例如日志记录尚未发生的事、陌生地点出现 Camper 旧物、异常身影似乎认识 Camper、不同地图出现相同符号、已丢失物品在别处出现、NPC 坚称见过当前队伍，或重复事件里的台词细微变化。

暗线不得阻碍基础玩法：不完成暗线也能解锁新地图，不阅读日志也能正常冒险，不收集全部线索也不会卡住物品系统，错过稀有事件不会永久失去推进机会。

## 深山白影

白影是暗线的第一个入口，但不是强制调查目标。表面解释可以是雾中人影、动物错觉、护林员传闻、山林异常或鬼影。隐藏方向可以与 Camper 曾经经历、未来经历或没有发生的其他冒险有关。

当前不要在代码或玩家文案中确认白影一定是未来 Camper、平行世界 Camper、鬼，或证明世界存在时间循环。必须保留解释空间。

未来可逐渐让白影背着相似背包、使用 Camper 曾携带或丢失的物品、说出口头禅、对特殊物品有熟悉反应、在迷路时引路或误导。

## 地图规划

未来主要地图可能包括：

```text
深山 → 雾雨林 → 沉没海域 → 火星旧基地 → 银河区域
```

这不是玩家界面必须显示的强制章节顺序。每张地图预计包含 3-4 条路线、15-25 个基础事件、3-5 条当地连续事件链、1 个主要环境机制、一批当地物品与收藏、少量跨地图暗线内容。解锁新地图后仍应能返回旧地图，新地图物品、队友或经历也能改变旧地图事件。

## 数据生命周期

长期 adventure 存档容纳：

```js
adventure: {
  stamina,
  storage,
  unlockedMaps,
  mapStates,
  adventureMemories,
  discoveredClues,
  recentAdventureHistory,
  lastLog,
  pendingBackpack,
  pendingLoot
}
```

地图当地状态属于地点。深山当前记录 `bridgeRepaired`、三阶段 `cabinSearched`、`rangerCluesFound`、`animalTrust`、`whiteShadowTrust`、`animalEncounters`、`whiteShadowEncounters` 和逐事件 `recurringEncounters`。它决定场景变化、路线开启、当地 NPC 或异常是否见过玩家、重复事件阶段和后续结果；所有计数与信任值都有边界。

Camper 跨地图记忆跟随角色延续。当前只保存真正可跨地图复用的 `rescuedSomeone`、`supernaturalEncounters`、`seriousFalls`、`animalTrust`、`frightenedByApparition`、`canopyCrossings`、`stationRecordsRecovered`、`sharedSymbolEncounters` 和 `preferredTools`，不把每个普通事件永久化。它影响相似危险中的谨慎程度、异常熟悉感、救援倾向、动物信任和新地图特殊气泡。后三个计数分别概括雨林树冠通行、调查站记录恢复和共同符号观察，不保存每条普通事件明细。

本趟临时状态只属于当前 trip：

```js
trip: {
  mapId,
  routeId,
  adventureHook,
  backpack,
  loot,
  eventFlags,
  storyBeats,
  participants,
  mapStateSnapshot,
  adventureMemorySnapshot,
  eventIndex,
  staminaStart,
  status
}
```

`eventFlags` 和 `storyBeats` 不能错误继承到下一趟。出发时读取的地图状态与 Camper 记忆保存为本趟内存快照，冒险中途不会因结算顺序改变正在发生的故事；结束后才把长期地图变化写入 `mapStates`，把少量跨地图经历写入 `adventureMemories`。

冒险存档当前为 v8。v7 及更早日志缺少的队伍、分工、个人观察和高光字段会在 sanitize 时补为空值或单人默认值；v6 及更早存档会补齐三个跨地图经历计数，缺失值为 0；v5 及更早存档仍会补齐第二张地图状态和新增物品规则。旧 `cabinSearched: true` 迁移为阶段 1，已有桥梁、线索、信任、仓库、体力、背包、loot、解锁地图与近期历史均保留，越界值会收敛到合法范围，不重置整个 adventure 存档。

## 冒险契机

不做手动任务目标。系统根据地图、路线、未完成线索、上次结果、地图状态、Camper 记忆、近期历史、特殊物品和少量随机性自动生成 `adventureHook`。准备页只显示一条不可点击、不需要确认的只读提示；更换路线时可以重新生成适合该路线的契机。

`adventureHook` 是叙事契机，不是传统任务。它只负责轻度提高相关事件权重、帮助组织 1-3 个相关事件、给日志一个主题，并生成“有所发现”“线索继续”“暂无结果”或“意外偏离”的结尾。它不固定五个事件，不保证奖励，不显示成功条件，未推进也不算失败；它不得成为暗线推进或地图解锁的强制门槛。出发时 hook 以 `{ id, title, intro, relatedEventIds, source }` 快照写入 trip，途中不随地图状态变化。

## 物品流转

当前规则保持：

- 出发背包最多 5 件。
- 鱼、料理和冒险物品统一占格，同种多个分别占格。
- 本次发现进入独立 `loot`，不占出发背包容量。
- 冒险结束后按来源返还：鱼回鱼、料理回料理、冒险物品回 adventure storage。
- `pendingBackpack` 与 `pendingLoot` 用于刷新和中断保护。
- 工具和消耗品由系统按规则自动使用，玩家不逐件设置权限。

## 未来组队

长期计划支持单人、2 人、最多 4 人。当前本地事件引擎已验证单人、2 人和最多 3 人；第 4 人只保留数据上限，不提供当前模拟入口。多人不是简单叠加 traits，同一事件中不同 Camper 可以发起行动、使用自己的物品、帮助队友、发现不同细节、产生独立观察并获得个人高光。

事件需要允许记录：

```js
participants
actorCamperId
helperCamperIds
itemOwnerId
contributorIds
decisionSource
```

`decisionSource` 可以是 `auto`、`player`、`teammateSuggestion` 或 `timeoutAuto`；当前本地模拟统一使用 `auto`。行动者按事件反应与个人 traits 加权选择，并用历史行动次数平衡镜头；反应和结果读取行动者自己的快照，不叠加团队 traits。协助者最多 2 人，物品所有者可以与行动者不同，贡献者和个人观察分别记录。当前不实现多人 UI、好友邀请或联机后端。

## 当前已实现

- 深山单人冒险原型。
- 5 件背包，鱼、料理、冒险物品统一占格。
- 独立 loot 与结算返还。
- 冒险体力消耗和随时间恢复。
- 深山 4 条路线、5 个后台叙事契机、22 个基础事件，每趟最多 5 个不重复事件。
- 根据路线、深山状态、近期历史、线索、携带物与少量随机性自动选择 `adventureHook`。
- 准备页只读契机提示，以及故事化的“有所发现 / 线索继续 / 暂无结果 / 意外偏离”日志。
- 基于 Camper traits、互动习惯、每日运势、携带物品和随机性的自动反应。
- 工具专属反应、消耗/丢失/获得提示、唯一物品规则。
- 深山连续事件临时 flags、故事型日志、路线解锁记录。
- 深山长期桥梁状态、三阶段木屋搜索、护林员线索、动物/白影信任与相遇次数。
- 吊桥、木屋、动物和白影的首次/再次/多次相遇气泡与结果变化。
- 动物、白影、护林设施、安全路线、溪流旧设施、瞭望塔信号共 6 条可贯通的当地事件链。
- 黎明晨雾、山脊横风、夜间回声与既有暴雨/浓雾环境变化。
- 12 种既有冒险物品在新增事件中的多处用途，以及材料、工具、消耗品、线索与路线解锁的完整回流。
- 未结束地图经历对自动契机的优先级，以及同路线近期事件、位置组合和已结束契机降权。
- 有边界的 Camper 跨地图记忆与最近使用工具顺序。
- `pendingBackpack` / `pendingLoot` 刷新和中断保护。
- Adventure Storage 入口、手机端布局、控制台测试入口。
- 注册表驱动的地图选择页；深山可进入，雾雨林在内容完成前显示低剧透锁定预览。
- 从准备页和日志页返回地图选择；解锁新地图后旧地图仍保留入口。
- 每张地图独立的 `mapStates`，以及跨地图共享的体力、背包、Storage 与 Camper 记忆。
- 通用 `unlockMap` 结算和地图场景/路线/事件/Hook 注册接口，新增地图不复制深山状态机。
- 可进入并可返回的第二张地图“雾雨林”，包含河岸湿地、藤蔓密径、树冠旧路和废弃调查区 4 条路线。
- 雾雨林 4 个自动叙事契机、18 个基础事件和湿地补给、发光植物、变化藤蔓、调查站记录 4 条当地连续事件链。
- 雾雨林本趟 `rainforestMoisture` 湿润度机制；它只存于当前 `eventFlags`，轻度改变水位、孢子与树冠事件倾向和故事文案，不向玩家显示数值。
- 雾雨林独立当地状态：`trailSaturation`、`vineRouteState`、`stationResearchStage`、`insectPressure`、`wetlandKnowledge`、`luminousPlantStudies` 和逐事件重复次数。
- 藤切刀、驱虫叶露、微光孢子、压叶标本、调查站通行牌和苔纹雨披 6 种当地物品；其中消耗品可重复补充，收藏和工具遵守唯一持有规则。
- 雾雨林由普通冒险次数或深山护林员当地故事解锁，不依赖暗线；地图开放后两条入口路线自动可走，进阶路线继续由当地事件解锁。
- 雾雨林独立背景、前景遮挡、4×3 事件物件图集和 3×2 当地物品图集。
- 深山的严重跌落、救援、异常相遇和动物信任会在雨林树冠、信标、三叶石纹/错误记忆空地和蛙鸣事件中生成跨地图气泡与结果变体。
- 雨林结算会把树冠通行、调查站记录恢复和共同符号观察写入共享 Camper 记忆；这些经历返场后会改变深山吊桥、护林笔记和发光菌环的观察方式，但不修改深山 `mapState`。
- 雨林藤切刀和苔纹雨披可实际携回深山，分别为废弃套索和暴雨遮蔽提供专属解法；两件物品仍只来自雨林当地池。
- 深山三叶/菌环、雨林石纹与错误记忆空地形成可关联但可完全忽略的暗线细节；玩家文案不使用“回途”名称，不确认两地异常具有单一答案。
- 纯本地 1–3 人事件分工原型：按性格与事件匹配选择行动者，五事件内平衡镜头，分配 1–2 名协助者，并记录物品所有者、贡献者和自动决策来源。
- 深山与雾雨林各有 3 个事件提供队员独立观察规则；高观察、高社交或拥有相关记忆的 Camper 可以看到不同细节，团队日志保留各自观察。
- 每位本地队员生成行动、协助、物品贡献和个人观察高光；`simulateLocalAdventurePartyTest()` 可在 Console 运行不写存档的本地模拟。
- 真实联机所需权威 trip、背包所有权、幂等结算、库存账本、异步恢复和权限边界记录在 `ADVENTURE_MULTIPLAYER_API_REQUIREMENTS.md`。

## 当前尚未实现

- 真实异步多人组队、好友邀请、多人日志 UI 和联机后端。
- 当前本地模拟的第 4 名成员、真实队伍准备流程和公共 loot 分配规则。
- 完整暗线剧情、白影身份揭露或银河结局。

## 当前阶段边界

深山与雾雨林已形成双向返场，本地 1–3 人事件原型也已在同一事件日志结构上验证行动者、协助者、物品所有者、个人观察和高光。最终阶段只做新旧档、双地图往返、物品与中断、离线体力、移动端和完整语法回归，不接真实网络、好友系统或多人 UI。
